## 🧭 فلسفة المشروع

Noorify ليس مجرد بوت…  
إنه **نظام رقمي لخدمة الذكر والدعوة**، لذلك نركز على:

- 🧼 **نظافة الكود**
- ⚡ **الأداء العالي**
- 🧠 **البساطة الذكية**
- 🔁 **قابلية التوسّع**

أي مساهمة يجب أن تحترم هذه المبادئ.

---

## 🚀 البدء السريع (Getting Started)

### 1️⃣ عمل Fork للمشروع

```bash
git clone https://github.com/YOUR_USERNAME/Noorify_Bot.git
cd Noorify_Bot
````

---

### 2️⃣ تثبيت الحزم

```bash
npm install
```

---

### 3️⃣ تشغيل المشروع

```bash
npm run dev
```

---

## 🧱 أنواع المساهمات (Contribution Types)

| النوع         | الوصف            |
| ------------- | ---------------- |
| ✨ Feature     | إضافة ميزة جديدة |
| 🐛 Bug Fix    | إصلاح مشكلة      |
| ♻️ Refactor   | تحسين الكود      |
| 📚 Docs       | تحسين التوثيق    |
| ⚡ Performance | تحسين الأداء     |

---

## 🔀 نظام الفروع (Branching Strategy)

| الفرع       | الاستخدام       |
| ----------- | --------------- |
| `main`      | النسخة المستقرة |
| `dev`       | التطوير         |
| `feature/*` | ميزة جديدة      |
| `fix/*`     | إصلاح           |

### مثال:

```bash
git checkout -b feature/tasbih-ui
```

---

## 📥 خطوات إرسال مساهمة (Pull Request Flow)

1. Fork المشروع
2. إنشاء فرع جديد
3. كتابة الكود
4. اختبار التعديلات
5. إرسال Pull Request

---

## 🧾 معايير قبول المساهمة

### ✅ يجب أن:

* الكود نظيف ومفهوم
* لا يكسر أي ميزة حالية
* يحتوي وصف واضح في PR

### ❌ سيتم رفض:

* كود عشوائي
* تعديلات بدون اختبار
* Feature غير واضحة

---

## 📌 قواعد كتابة الكود (Code Standards)

### 🧼 1. نظافة الكود

* أسماء واضحة (camelCase)
* دوال قصيرة
* لا تكرار (DRY)

### 🧠 2. تنظيم المشروع

* فصل الـ Logic عن الـ Commands
* استخدام Modules
* عدم وضع القيم مباشرة (Hardcoded)

### ⚡ 3. الأداء

* تجنب العمليات الثقيلة داخل loops
* استخدم caching عند الحاجة

---

## 🧪 الاختبار (Testing)

قبل إرسال PR:

```bash
npm run test
```

إذا لا يوجد Tests:

* اختبر يدوياً
* تأكد عدم وجود Errors

---

## 🧩 أسلوب الرسائل (Commit Style)

```bash
feat: add tasbih progress system
fix: resolve search bug
refactor: improve database handler
```

---

## ⚠️ أخطاء شائعة

* ❌ إرسال PR ضخم يحتوي عدة تغييرات
* ❌ تجاهل هيكل المشروع
* ❌ عدم اختبار الكود
* ❌ كتابة كود بدون شرح

---

## 💡 اقتراح ميزة (Feature Request)

افتح Issue واكتب:

* المشكلة الحالية
* الحل المقترح
* مثال عملي

---

## 🐞 الإبلاغ عن مشكلة (Bug Report)

يرجى تضمين:

* وصف المشكلة
* خطوات إعادة المشكلة
* Logs إن وجدت
* لقطات شاشة

---

## 🛡️ الأمان (Security)

🚫 لا تقم برفع:

* Tokens
* API Keys
* ملفات حساسة
